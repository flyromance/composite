  • 性能优化: diff算法
    • 操作dom是昂贵的，随便打印一个dom，特别大的对象，vdom很小，浏览器运行js特别快，肯定比dom快
    • 另一方面，js操作dom，渲染进程得响应，就是线程直接的交互，少交互为好。于是出现了diff算法
    • 比较新旧vdom，找出差异patches，再修改旧的dom
    • 或者比较旧的dom与新的vdom的差异，直接修改旧的dom，少一次递归过程
  • 性能优化: 异步setState
    • 合并state，不能每次修改都
    • 异步执行flush，涉及到了event loop的知识。
  • 性能优化: shouldComponentUpdate
    • 因为一个引用对象，修改了其属性，数据变了，但是比较的时候，是相等的！
    • 于是出现了immutable, 不可变数据
